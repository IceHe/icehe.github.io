子问题：
PHP中 !== 的逻辑含义


var === NULL 的意思是：
变量var 无论类型，还是值，都等于NULL。
var !== NULL 的意思是：
变量var 无论类型，还是值，都不等于NULL。

设 type 的意义为 var 的 类型等于 NULL
设 value 的意义为 var 的 值等于 NULL
（ 以下 type、value 简写为 t 和 v ）

则 var === NULL 的逻辑表达式
是 type && value
则 var !== NULL 的逻辑表达式
是 ! type || ! value 
也可写为 ! (type && value)
（上面 5 行的结论，有PHP程序的运行结果作支持：
     相关笔记记录 PHP empty() 和 isset() 的区别 ）

所以，! (var === NULL)
等价于 var !== NULL

=======================
引申：
var == NULL 的逻辑表达式是 value
var != NULL 的逻辑表达式是 ! value
! (var == NULL) 等价于 var != NULL
=========================

isset(var) 的意思是：
检测变量var 是否设置，并且不是 NULL。
若 var 存在，并且值不是 NULL，则返回 TRUE，
否则返回 FALSE。

设 exist 的意义为 变量 var 存在
则 isset(var) 的逻辑表达式
是 exist && var !== NULL 
也可写为 exist && ! ( var === NULL ) 
即 exist && ! ( type && value )
即 exist && ( ! type || ! value )

所以，严谨来说，isset(var)
不等价于 var !== NULL
也
不等价于 ! (var === NULL)

不过，exist这个变量在某些情景下，
与该具体问题无关时，在编程时，
可以用 isset(var) 代替 var !== NULL。


关键结论：
var===NULL     type && value
var!==NULL     ! type || ! value
isset(var)          exist && (! type || ! value)

! (var===NULL) 等价于 var !== NULL
isset(var) 不完全等价于 var !== NULL
