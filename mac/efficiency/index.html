Mac Efficiency 效率指北

2017.03 | Mac |
我这个「工具控、键盘党」如何使用 Mac。
  • 《Mac Shortcuts 快捷键》列举了绝大多数我用的快捷键。
  • 《Tools 利器》列举了我绝大多数我用的 Apps。
  • 不同于上文，本文说的是对「如何高效地使用 Mac」的个人见解和建议，仅供参考。
    • 有部分叙述与《Tools 利器》重复。
    • 本文会列举不少快捷键，但不会所有的都说。因为它们只是我为了引出想说的东西而举的例子。
    • 我一开始也是 Windows 用户，后来才用 macOS，所以本文会说到 Windows 和 macOS 之间的差异和对比。
    • 除非有额外的说明，否则默认都是 macOS 上用的快捷键。

好处

  • 原来不习惯使用快捷键的人，和刚转到 macOS 平台用惯 Windows 快捷键的人，难免要学习、重新适应，不过这是值得的。熟悉一些基本的快捷键，操作效率就能显著提升。
    • 用上瘾之后，可能甚至恨不得连触控板都不用了（走火入魔）…… 显然我是尝试过的，感觉并不好。触控板的操作更符合直觉，几乎没有学习成本。
      （而试图用键盘完全取代触控板的 Apps，要不学习曲线不平缓，很难适应，否则还不如直接用触控板操作来得快，而且手指的操作压力也很大，长时间敲键盘为生的人手指手腕是会痛的。）
  • 先熟悉通用性强、常用的 macOS 系统快捷键。
    绝大多数第三方 Apps 都会遵循系统快捷键的设计逻辑和设置规律，复用大多数常用的快捷键，降低学习新 App 快捷键的成本，以便用户使用。
  • 再熟悉常用 Apps 的常用快捷键。
    日常使用的 Apps 一般就没几个，可以参考二八法则，只学习其中 20% 常用 Apps 的 20% 常用快捷键，就能以较低的学习成本大大提升效率。
  • 继续进阶，你可以改键盘的键位，使用增强功能的 Apps。
    • 修改不合理（你认为的）的键盘键位。比如我也认同「大写锁定」功能不值得占据 ⇪ 这个好按的键位，中文使用者就很少输入大写字母，连问多英文使用者也会修改这个键。
      • 可能你适应了新键位之后，你用别人的电脑或者别人用你的 Mac 都会不方便，但是你想想你是不是正常情况下 98% 都只使用自己的设备，所以这么提升效率是值得的！
    • 许多 Apps 可以增强系统的功能，比如防止 Mac 睡眠、调整窗口布局、截图录屏、科学上网、隐藏菜单栏无用的图标等，总有一款能解决你的痛点。

macOS

Ref: 《 Mac 键盘快捷键 》

Default 默认

基本的 zxcv

剪切、复制、粘贴、撤销、重做

  • 剪切、复制、粘贴：最高频的操作，Windows 上是 ^ x、^ c、^ v；而 macOS 上则是 ⌘ x、⌘ c、⌘ v。
    • 刚从 Windows 切换到 macOS 时会不适应。习惯之后就会发现 ⌘ 在空格旁边，离大拇指很近，它相关的组合快捷键其实比 ^ 的敲起来舒服多了。
  • 撤销、重做：不同于 Windows 用 ^ y 来进行「重做」（即取消 ^ z 的撤销操作）；而 macOS 用 ⌘ ⇧ Z。
    • 当你 macOS 的快捷键用多了，会习惯猜测一个操作对应的快捷键组合额外加上 ⇧ 后，就能触发相反的操作。正如上文的 ⌘ ⇧ Z 之于 ⌘ z ，又如 ⌘ ⇧ ⇥ 之于 ⌘ ⇥（即「正向/反向切换 Apps」）。
  • 启示：「相关快捷键之间存在内在的逻辑规律和关联」是个好设计。
    • 它能促进用户对「某一操作该对应哪个快捷键组合」产生一种直觉，用户可以籍此更容易猜出想要的快捷键在哪，大大减轻记忆快捷键的负担。
      • 比如，上述的「快捷键组合加 ⇧ 能触发反向操作」的规律。
    • 还有 某操作所用的快捷键组合，通常跟该操作的英语（单词）表述有关：
      • 比如，以上 ⌘ c 的 c 即 copy，⌘ x 的 x 像剪刀，也能跟 cut 扯上关系。
      • 快捷键组合确实有这样的规律，但是也不总是这样。比如，⌘ v 的 v 跟 paste 就没有明显的关联。题外话：当然你仍然可以说 v 像插入内容的标识，或者说它暗示 verbatim（逐字的、一字不差的），实际上这两个说法太 afterism（后验主义）了。
      • 为什么「撤销、剪切、复制、粘贴」的快捷键对应 zxcv 等键位？最显而易见的原因 是「zxv 离 c copy 很近，这些操作互有关联，适合放到一起，敲起来更方便。」

文本编辑

  • 基本：⌘ b 加粗、⌘ i 斜体、⌘ u 下划线，⌘ = / ⌘ - 放大/缩小字体（或界面的显示）……
  • 进阶：
    • 选中 文本：（Selection）
      • 按住 ⇧ 不放，再按 ↑↓←→ 等方向键，
    • 删除：
      • Fn ⌫ 后向删除字符，跟 ⌫ 前向删除相反。
      • ⌥ ⌫ 前向删除一个（中英）单词（或字符串），当然也有对应的 Fn ⌥ ⌫ 后向操作。
        注意 macOS 对中文单词的判定并不总能合乎你的期待。
    • Emacs：
      系统内置文本编辑控件内置部分 Emacs 式的编辑快捷键。
      • 跳转：^ a 即 Windows 的 Home，^e 即 End。
      • 删除：^ k 即删除光标当前位置至行尾处的字符。
      • 方向：^b 即 ←，^ f 即 → 等。

文件、目录

移动、删除、预览、打开、向上、查看文件属性

  • 文件移动：在 Windows 先用 ^ x 剪切，再用 ^ v 粘贴；而 macOS 则先用 ⌘ c “复制”目标文件，再在目标目录下用 ⌘ ⌥ v “粘贴”，此时文件「移动」成功！……
    • 如果你习惯使用 Windows 后来才转用 macOS，可能会不适应 macOS Finder（文件管理器）移动文件的思路。(只有文件移动才会这样…奇葩)
    • 解释：先用 ⌘ c 并非一定表示「复制」的操作，也可能是「移动」，具体是哪一种操作，取决于后续用哪个快捷键；如果后用 ⌘ v 即是「复制、粘贴」，⌘ ⌥ v 则是「剪切、粘贴」即「移动」（你会发现这两个快捷键也是关联的，并不难记）。
    • 进行文本操作时，还是用 ⌘ x 剪切，用 ⌘ ⌥ v 会显示最近的剪切复制的文本历史，可以选择其中一个进行粘贴。
  • 删除：⇧ ⌫，只是进入废纸篓；完全删除则需要清空废纸篓，或者用命令行进行删除操作。
  • 预览：选中文件，然后按空格键 凵 即可便捷地预览内容，这样一般情况下就用不着专门启动一个 App 来查看文件了，除非需要修改。配合 预览插件 效果更佳。
  • 打开：Windows 上用户会习惯用 ↩ 来打开文件和应用；然而这样做在 macOS 则是 重命名，如需打开要用 ⌘ ↓。
  • 向上：即跳转到上一层目录 ⌘ ↑。
  • 查看文件属性：⌘ i 通常打开这个「文件属性」信息框，是为了 —— 修改这类文件默认用什么 App 打开，修改 App 的图标、获取其图标等。
  • 不常用/非必需：
    • 文件复制：macOS 用 ⌘ d 即「制作副本」，等于在同一目录下先用 ⌘ c 再用 ⌘ v。
    • 制作「替身」：你可能会将一些文件复制多份，然后放在不同地方，可能是为了便于访问。但是除非为了备份，否则这么做会令你难以对其进行有效的「控制版本」。
      • 还不如用 ⌘ l 为其制作一个「替身」，将其放到其它地方，例如桌面。「替身」用程序员的话来说，就是指向源文件的「指针」或者「软连接」。然后打开替身等于打开了原始文件，可以进行编辑；删除「替身」也不会影响到原始文件。

其它常用

  • 设置：⌘ ,，我印象中没碰到过不默认使用它来打开设置选项的 App。
  • 窗口：⌘ ^ f 最大化，⌘ h 隐藏（相比之下 ⌘ m 最小化就没用了），⌘ ⌥ h 隐藏当前窗口之外的所有窗口（很好用）。
  • 浏览器：⌘ [ 前进、⌘ ] 后退。在 Finder 中也能用。
  • 输入法：^ 凵 切换输入法；注意系统自带输入法是用 [ 和 ] 进行候选字前后翻页的，而非一般的 , 和 .。
  • 关机：记住 ^  关机就够用了，还有 ^ ⇧  屏幕睡眠、⌘ ⌥ （硬盘）睡眠。
    macOS 虽然越来越不稳定了，需要强制重启来恢复系统的崩溃（只是我的情况，毕竟我经常折腾操作系统）。
  • Spotlight：习惯使用 ⌘ 凵 进行系统全局搜索，就不用总是打开 Finder 和 LaunchPad 来找文件和 Apps 了。
  • 功能键：即键盘最顶上的一排按钮（2016 年底新款 MacBook Pro 取消这排实体按键），控制「屏幕亮度、音量、音乐播放」等按键较常用，「任务控制（F3 处）、LaunchPad（F4 处）」等可以由触摸板的手势「三指上划、拇指加另外三指内划」来代替（详见 系统设置 → 触摸板 处）。
  • 除去不想提的快捷键，没有想起来的那些，一般也都不太重要，在此略过。

Custom 自定义

  • 在 系统设置 → 键盘 → 快捷键 中可以修改很多（不是全部）系统默认的快捷键。
    • 主要是将一些你觉得不合理的快捷键组合改得顺手，或者将用不上的快捷键去掉，以免误触。
      • 特别是对某些快捷键组合已经形成根深蒂固习惯的人，会记不住新的快捷键，经常按错。
    • 其中左侧候选栏中的 应用快捷键，可以用来重写全部或指定应用的某些快捷键。

浏览器

作为最最常用的 App，最值得熟练使用快捷键提升效率。

常用通用快捷键

  • 恢复（重新打开）最近关闭的标签页：⌘ ⇧ T，很有用的快捷键。
    特别是误关某些网页时，可以快速恢复而不用去浏览历史找。
  • 页面浏览：刷新 ⌘ r（Windows 则用 F5 刷新）；
    凵 向下滚动一页，⇧ 凵 相反；↑↓←→ 也可以小幅度滚动页面。
  • 访问历史：⌘ [ 前进，⌘ ] 后退，详情 ⌘ y。
  • 切换标签页（Tab）：^ ⇥ 下一个（右），^ ⇧ ⇥ 上一个（左）；
    ⌘ 1~9 跳到第 1~9 个标签页，如果一共超过 9 个标签页，⌘ 9 则是跳转到最后一个而非第 9 个。
  • 显示比例：⌘ = 放大，⌘ - 缩小，⌘ 0 恢复正常（默认）。
  • 收藏夹：⌘ d 添加到收藏夹，⌘ ⌥ b 管理收藏夹。
  • 其它：⌘ , 设置，⌘ ⇧ H 主页……

命令行

程序员相关。
工具链推荐：Zsh + Vim + tmux + iTerm 2

  • Zsh
    可能 Fish 配置、使用更友好一点点，但是它对 Bash 的兼容性不够好，Bash 的脚本或指令组合不一定能在 Fish 上跑，相反亦然，影响通用性。Zsh 在这方面则基本没有问题。综合各方面进行对比，我认为 Zsh 暂时是各种 Shell 中最好的选择。
    • 快速切换目录：最常用最便捷的功能
      • 直接输入目录的路径，毋需添加 cd 指令，即可切换到指定路径。
      • 输入不完整路径（或者指令）然后按 ⇥，自动补全目录（或指令），或者列出相似的候选项。
      • 输入 - 或 1 命令切换到上一个目录，1 ~ 9 则是回到之前第 n 个访问的目录。
    • Shell 内置的 Emacs 式快捷键（keybindings）
      • 跳转：^ a 光标移至行首，^ e 移至行末。
      • 删除：^ u 删除光标当前位置至行首处的输入，^ k 相反，即删除至行末；
        ^ w 删除前一个单词，^ h = ⌫ 删除前一个字符。
      • 移动光标：⎋, b 移动到上一个单词，⎋, f 移动到下一个（meta b / meta f for Bash）。
      • 命令行历史：↑ 上一条命令，↓ 下一条（^ p / ^ n 也可以）。
      • 确认：^ h = ↩ 确认指令。
    • 命令别名（Aliases）
      • 通常用来给常用命令指定一个「缩写」，一遍更便捷地输入长的命令组合。作为程序员，我的手速、输入准确率都一般，就特别喜欢输入缩写命令。
        • 例如，我让 gs = git status，gp = git push 等（见 Git Aliases 和我的 .zshrc 配置）。
      • alias 命令，列出所有的 aliases。
    • Zsh 还有许多好东西，这里暂略，可以根据自己的需求，自行查阅学习。
      • 可以简单参阅我的 .zshrc 配置。注意以上特性并不都是 Zsh 特有的，详看 各种 Shell 的对比。
  • Vim（也可以选择 Neovim）
    • 首先，通用性有保证！服务器没有 GUI 界面，可能没有 Emacs，但是基本不可能没有 Vi*，学会 Vim 保证你在几乎任何服务器环境下都能无碍地编辑文本。
    • 其次，拓展广。很多 IDE 和编辑器都有 Vim 插件，不用记太多各个 Apps 专有的快捷键，学会 Vim 就可以在其中进行绝大多数常用的文本编辑操作，减轻记忆负担，提升效率。
      • 例如，Sublime Text 系列就自带 Vintage Mode 专门支持 Vim 键位的操作。
    • 各种 Shell（命令行）除了自带 Emacs 式的快捷键，其实也有 Vim 式的快捷键，可以尝试一下，看看自己习不习惯。
  • tmux
    终端多路复用软件，即命令行中的「窗口管理」、「分屏工具」。
    • 允许一个用户在一个终端窗口或一个远程终端会话中，使用多个终端会话。
    • tmux 由 screen 发展过来，许多命令都是类似的。screen 虽不如后来者 tmux 易用、强大、美观，但是更通用，大多数服务器都有 screen 命令。
  • iTerm2
    功能、可配置性比系统自带的终端 Terminal 强大许多。对命令行的使用环境要求不高的话，用 Terminal 也没差，只要适合自己，够用就好。

全局进阶

  • Karabiner
    • 改键：
    • 我将 ⇪ 改成 ^。这么敲跟 ^、⌘ 组合的快捷键会顺手很多。
    • 原来的 ^ 键则用于触发 ^ 凵 以便一键切换输入法。
      • 用专用的单键 ^ 比用 ⇧ 或 ⇪ 来切换输入法，更不容易影响正常的操作。特别是用 Vim 输入中文时，因为 Vim 有太多跟 ⇧ 相关的操作。
      • 我用右手掌小鱼际（即靠近小指的部分手掌）去按原来的 ^ 键位，觉得比较方便，这样的话双手就基本不用远离键盘中心区了（参考盲打键盘的标准姿势）。
    • 我从键盘上完全移除了 ⇪ 的大写锁定功能（不改其它键来实现该功能）。
      • 因为很少用到，Vim 党可以先以小写写完，在用 U 指令全部改写为大写，大不了全用 ⇧ 来输入大写字母。
    • 右 ⌘ 改成 ⎋。因为左小指挪到键盘左上角的 ⎋ 略远，
      • 正如上文所说，这样的话双手就基本不用远离键盘中心区了（参考盲打键盘的标准姿势）。
      • 而且等于让比较强健的右大拇指去代替比较弱的小拇指（高强度使用经常会疼）去按 ⎋。
  • Manico
    • 快速启动、切换 Apps
  • Keyboard Maestro
    • 很不错，可以代替很多软件。
      • 脚本软件（AppleScript 的基本操作）
      • TextExpander
      • 快捷键设置软件！如 Manico。
        • 如：快速切换或打开软件
          • 通过简单的条件判断，可以设置开启条件、场景等等
          • 例如网络、蓝牙、相关App开启状态等相关条件
  • Karabiner + Seil
    • 可以该键位，让键位更合理，用更简单的动作来操作
      • 一个按键来切换输入法。
      • Esc、Ctrl 位置调整！
      • 数字键、功能键、方向键、符号键等的调整
    • （注意：暂时不支持 macOS Sierra 及往后的版本。新一代开发中 https://github.com/tekezo/Karabiner-Elements6VGi）
  • Spectacle | Moom
    • 窗口大小调动！
  • HyperSwitchy
    • 程序窗口切换！
      • 系统级别的不好用，简直有 Bug
  • Bartender 2
    • 简洁的工具栏
  • ShowyEdge
    • 直观查看输入法状态

  • HazeOver
    • 给非活跃的 App 窗口打上阴影！
  • Alfred？
    • 我用不上，毕竟我会自己写 AppleScript 脚本等等，直接调 快捷键去触发它们就好了。
    • 或者将脚本封装成 macApp 用 Spotlight 就能迅速匹配到该 App，然后迅速启动。
    • 我这个效率老鸟，真是想不到自己什么场景需要 Alfred 了？（希望可以有其它人告诉我这个答案）
  • Quick Look Plugins
    • 按空格键，可以预览看图片、文本等等，比较方便，不一定非要用 App 打开才能看到内容。
  • AppleScript
    • 进阶：直接操控操作系统的事件、配置、窗口，它必须学。
    • 其实也你一般没必要折腾，一般就有相关的 App 帮你解决问题，总会有办法的。
  • Automator
    • macOS 的按键精灵，有时候写 jQuery、JS 脚本不方便，Shell、Python、PHP 以致 AppleScript 写起来都麻烦，不如直接手动做。
    • 这是若只是重复性的简单操作，可能还不如直接 Automator 直接录制键盘、鼠标的操作来呢！
